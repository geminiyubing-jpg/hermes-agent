2. 核心模块详解

系统由6大核心模块组成，各司其职又紧密协作。点击标签页查看详情。

🧠 AIAgent 主引擎 run_agent.py

系统的核心调度器，管理完整的对话生命周期。

# 核心职责 class AIAgent: - 管理对话循环 (conversation loop) - 路由到正确的 LLM 提供商 - 处理工具调用请求和结果回传 - 管理迭代预算 (max turns: 90) - 自动上下文压缩触发 - 错误恢复和重试机制 - 流式/非流式响应处理 - 支持 Auxiliary Client (辅助模型)
Prompt Builder

动态组装系统提示词：角色设定、可用工具描述、用户画像、上下文注入、技能提示。

Model Metadata

模型特定的配置参数：Token估算、上下文窗口限制、工具调用格式适配。

🗜 Context Compressor agent/context_compressor.py

当对话超过模型上下文窗口时自动触发压缩，使用辅助模型进行摘要。

# 压缩策略 1. 保留 system prompt + 最近的对话（头部和尾部） 2. 对中间历史使用 LLM 生成摘要 3. 智能分配 Token 预算 4. 保护重要消息不被压缩 5. 支持迭代压缩（超长对话多次压缩）
Context Engine 插件

通过 context.engine 配置可替换内置压缩器。插件从 ~/.hermes/plugins/ 自动发现。

🔧 Tool Registry tools/registry.py

所有工具的自注册中心，使用注册表模式统一管理。

# 工具注册机制 每个工具模块在加载时自动调用 registry.register(): - schema: JSON Schema 格式参数定义 - handler: 实际执行函数（支持 async/sync） - metadata: 名称、描述、所属 toolset - availability: 运行时可用性检查 # 工具集 (Toolsets) toolsets = { "web": [web_search, url_fetch, browser], "terminal": [shell_execute, file_ops], "creative": [image_gen, music_gen], }
💾 Memory Manager agent/memory_manager.py

统一的记忆管理接口，支持8种后端插件。

# 记忆系统架构 MemoryManager (统一接口) |__ prefetch() -> 会话开始时预取相关记忆 |__ store() -> 存储新记忆 |__ search() -> 语义搜索 |__ reflect() -> 定期反思总结 # 后端插件 Holographic | Hindsight | RetainDB | Honcho OpenViking | ByteRover | Mem0 | Supermemory
🌐 Gateway 网关 gateway/

多平台消息路由中心，统一处理来自不同聊天平台的消息。

# Gateway 核心流程 Gateway.run() |__ 加载平台适配器 (Telegram/Discord/Slack/...) |__ 为每个平台创建消息监听 |__ 消息到达 -> 路由到对应 Session |__ Session 管理 (创建/重置/清理) |__ 调用 AIAgent 处理消息 |__ 响应路由回原始平台 |__ 后台任务 (Cron执行 / 定时清理) # Session 策略 session_reset: daily | idle | both | none
🎭 Agent Orchestration agent_orchestration/

多Agent协调系统，支持子Agent的生成、通信和生命周期管理。

# 多Agent架构 Manager (线程池管理) |__ spawn_agent() -> 生成子Agent |__ 最大委派深度: 2层 |__ pending -> running -> completed/failed Mailbox Hub (消息传递) |__ 混合存储: 内存队列 + JSON持久化 |__ 消息类型: message / task / result / permission |__ 线程安全 + 跨进程安全 # 工作流模式 parallel | sequential | map_reduce | DAG

🌐 Web Tools

📁 File Tools

文件读写、搜索、目录操作、文件系统检查点。

⚡ Skills Tool

动态技能加载与执行，按需从Skills Hub安装。

🔗 MCP Tool

Model Context Protocol集成，连接外部MCP服务器。

🎨 Image Tools

图片生成、分析、处理（多模态支持）。

📝 Code Tools

代码执行、语法高亮、代码审查辅助。

⏰ Cron Tool

定时任务创建、管理、执行。

工具执行流程

1

LLM返回 tool_call

模型决定调用某个工具，返回工具名称和参数（JSON Schema格式）

▼
2

Registry查找

Tool Registry根据名称查找已注册的工具，验证可用性

▼
3

权限检查

根据toolset配置和用户权限，决定是否允许执行

▼
4

pre_tool_call 钩子

插件可在此拦截、修改参数或阻止执行

▼
5

工具执行

调用handler函数（同步或异步），获取执行结果

▼
6

post_tool_call 钩子

插件可在此处理结果、记录日志等

▼
7

结果回传LLM

工具结果作为消息添加到对话历史，LLM继续生成响应

9. 使用说明

从安装到日常使用的完整指南。

📦 安装部署

方式一：一键安装

# macOS / Linux curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash # Windows (PowerShell) iex (Invoke-RestMethod https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.ps1)

方式二：开发者安装

git clone https://github.com/NousResearch/hermes-agent.git cd hermes-agent uv venv venv --python 3.11 source venv/bin/activate uv pip install -e ".[all,dev]"

方式三：Docker

docker build -t hermes-agent . docker run -it --rm -v ~/.hermes:/root/.hermes -e OPENAI_API_KEY=your-key hermes-agent
⚙️ 初始配置

交互式配置向导

hermes setup # 将引导设置： # 1. 模型和提供商 2. 终端执行后端 3. Agent参数 # 4. 消息平台 5. 工具配置

手动配置

vim ~/.hermes/config.yaml vim ~/.hermes/.env # .env 示例： OPENAI_API_KEY=sk-... ANTHROPIC_API_KEY=sk-ant-... TELEGRAM_BOT_TOKEN=...
